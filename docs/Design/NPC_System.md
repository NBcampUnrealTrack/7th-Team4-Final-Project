## 1. 개요

NPC 시스템 MVP 설계 요약본.
다른 시스템과는 이벤트/인터페이스 기반의 느슨한 결합을 유지한다.

### 1.1 NPC 시스템 책임 범위

- NPC와의 대화(다이얼로그) 트리거 및 진행
- 퀘스트 수락 — 한 파티원이 수락하면 전원에게 동시 부여
- 인터랙션 시스템 연동 (F키 상호작용)

### 1.2 구현 방식

- C++ 베이스 클래스(`ANPCCharacter`) + Blueprint 파생 구조
- 퀘스트 수락은 서버에서 처리 — 파티원 전원에게 RPC로 전파
- 다이얼로그 / 퀘스트 시스템과 델리게이트(Delegate) 단방향 이벤트 통신

## 3. 대화 (다이얼로그) 시스템

플레이어가 NPC 범위 진입 시 `[F] 대화하기` 안내 표시 (인터랙션 시스템 담당).
F키 입력 → 다이얼로그 위젯 오픈 (UI 파트 연동).
대화 진행: Enter / Space로 다음 줄, 숫자키·클릭으로 선택지 선택.
대화 종료 시 퀘스트 수락 분기로 이어진다.

---

## 4. 퀘스트 수락 — 파티 공유

핵심 규칙: **한 파티원이 NPC에게 퀘스트를 수락하면 파티 전원이 동시에 해당 퀘스트를 받는다.**

### 4.1 흐름

```
플레이어가 퀘스트 수락 선택
  → 서버에서 처리
  → 파티원 목록 조회
  → 전원에게 퀘스트 등록
```

### 4.2 중복 수락 처리

| 상황 | 처리 |
|------|------|
| 이미 퀘스트를 보유한 파티원 | 중복 등록 없이 스킵 |
| 이미 완료한 파티원 | 완료 상태 유지, 재등록 안 함 |
| 파티 미결성 상태 (솔로) | 본인에게만 등록 |
